## 2. Структурные болячки

### 2.1. Нет `cmd/`
Бинари в `client/` и `server/` рядом с бизнес-кодом. Канон Go: `cmd/<bin>/main.go` = wiring only. Сейчас `client/` одновременно:
- путь к бинарю;
- родитель `client/internal/`;
- содержит 875-строчный `manual_captcha.go` в `package main`.

### 2.2. `manual_captcha.go` в `package main`
875 строк (больше самого `main.go`): HTTP-сервер на 8765, запуск браузера, gzip-rewrite VK-страниц, регекс-патчинг, capability detection. Тесты в `package main` — изолированно не тестируется. Должен быть отдельный пакет.

### 2.3. Двойная `internal/` иерархия
`client/internal/` vs `internal/`. Правило выбора **де-факто**: «если общее — наверх, если клиент-only — вниз». Сломалось на `internal/ish` — он клиент-only, но потребитель (vless) живёт в `internal/`, поэтому ish вытолкнут наверх. То есть правило перевёрнуто: «куда импортируется из общего, то наверх». Это симптом, не дизайн.

Раздробление:
- `client/internal/vkauth, captcha, browserprofile, dnsdial, namegen` — клиент-only.
- `internal/proxy/{udp,vless}` — по смыслу клиент-only, но в общем.
- `internal/ish` — клиент-only, но в общем (ради vless).
- `internal/bond/{frame,client,server}` — frame общий, client клиент-only, server сервер-only — три разных аудита смешаны.

### 2.4. `tcputil` top-level
Не `cmd/`, не `internal/`. Side-quest (переименовать в `kcptun`) не закрыт. Имя врёт: внутри KCP/smux профили, никакого TCP.

### 2.5. Side-quests не закрыты (из старого плана)
- `tcputil` → `kcptun` — не сделан.
- `selectedKCPProfile()` `os.Getenv` в hot path → `sync.Once` — не сделан.
- `isDebug` глобал → logger interface — не сделан.
- `globalAppCancel` убран (4.1), но `globalBondRegistry` остался (5.2).

### 2.6. Логирование — три способа
- `log.Printf` (stdlib).
- `debugf` (gated глобал в обоих main).
- `Deps{Debug bool, Debugf func}` callback в каждом пакете.

Один `internal/logx.Logger` решает.

### 2.7. `globalBondRegistry` инициализация на package init
`server/main.go:155`:
```go
var globalBondRegistry = bondserver.NewRegistry(bondserver.Deps{Debug: isDebug, Debugf: debugf})
```
Здесь `isDebug == false` (main ещё не выполнен). Работает только потому что bondserver **не читает** поле `Debug`, использует только `Debugf` callback. Bug-by-design, который сейчас не стреляет. Стрельнёт когда кто-то прочтёт `Debug`.

### 2.8. `proxy/udp` и `proxy/vless` — 70% общего кода
Оба:
- открывают N TURN-стримов через `turnpipe`;
- крутят DTLS через `dtlsdial`;
- ходят за VK creds (`vkauth.Client`);
- держат `activeLocalPeer` / `connectedStreams`.

Различие: что копировать поверх DTLS. Это **handler**, не отдельный mode. Нет общего `proxy.Engine` + handler-интерфейса.

## 3. Путаница в предметной области

### 3.1. Имена врут

| Имя | Что говорит | Что делает |
|-----|-------------|------------|
| `-wrap` (CLI help) | «ChaCha20-XOR obfuscate» | ChaCha20-Poly1305 **AEAD** |
| `wrap` (пакет doc) | «SRTP-mimicry AEAD» | то же |
| `wrap` (Go-термин) | `errors.Wrap`, `net.Wrap*` | здесь — VK-bypass обфускация |
| `tcputil` | TCP utils | KCP/smux профили |
| `turnpipe` | data pipe | TURN dial+allocate setup |
| `pipeConn` (server) | то же слово | **bidi copy loop** — другое! |
| `netadapt` | adapter | conn wrappers |
| `-no-dtls` ↔ `Direct` | один флаг | два имени между CLI и Go |
| `-turn` → `Host` | «TURN server ip» | принимает host (не только IP) |
| `-connect` (server) | error: «server address» | реально backend address |

### 3.2. Перегруженные термины

**«Client» — 4 сущности:**
- `cmd/client` бинарь
- `vkauth.Client` HTTP-клиент к VK
- `bondclient.Handler` наша bond-сторона
- `turn.NewClient` TURN-protocol-клиент

**«Bond» — feature или подсистема?** Layout `internal/bond/{frame,client,server}` делает его равным `wrap`. Реально bond — **только опция vless-режима**. Иерархия преувеличивает значимость.

**«Stream» — 3 значения:**
- TURN stream (одна из N TURN-connections)
- smux stream (внутри одной DTLS-сессии)
- VK credential stream / cache (одни creds на группу TURN-streams)

`-streams-per-cred` → `streamsPerCache` в коде. Грепаешь — путаешься.

**«Mode» — 5 ортогональных переключателей**, нигде не сгруппированы:
- `-udp`: TURN transport (UDP|TCP к TURN).
- `-vless`: app proxy (UDP-relay|TCP-forward).
- `-vless-bond`: внутри TCP-forward.
- `-wrap`: obfuscation (off|SRTP-mimicry).
- `-no-dtls`: DTLS off (DO NOT USE).
- `-dns`: udp|doh|auto.

Плоские bool в `config.Client` без группировки.

### 3.3. Концептуальные дыры

**a) DTLS = obfuscation, не security.** Доки, переменные, контекст говорят про DTLS как защиту. Реально DTLS нужен чтобы VK content-filter увидел legit-handshake — иначе drop. `-no-dtls` помечен «DO NOT USE» именно потому что не работает в проде. Это **нигде не зафиксировано**.

**b) Captcha разбит на 3 места:**
- `client/internal/captcha/` — auto slider-solver.
- `client/manual_captcha.go` (875 строк, `package main`) — manual flow (browser+http+gzip).
- `vkauth.AutoSolveFunc`/`ManualSolveFunc` — DI-точки.

Один домен «как пройти VK капчу», три места.

**c) `ish` package** — iOS sandbox shim. Имя расшифровывается только если знаешь iSH (Linux-под-iOS на iPhone). Без header-doc — мистика.

**d) Auto-detect bond на сервере** (`bond-autodetect=true` на старте) делает `server.-vless-bond` флаг **частично избыточным**. Документация не объясняет когда он нужен.

### 3.4. TCP/UDP терминология (специально вынесено)

**Два разных слоя:**
1. **TURN transport** — флаг `-udp`. Default `false` → TCP/TLS к TURN. True → UDP к TURN.
2. **App proxy mode** — флаг `-vless`. Default `false` → UDP packet relay. True → TCP forwarder (внутри VLESS-генерик).

«TCP-режим» в проекте **есть** — это `vless`. Имя историческое (первый потребитель — Xray VLESS). Внутри ничего VLESS-специфичного: tcp-байты в smux-стрим, на сервере `net.Dial("tcp", backend)`. Честное имя пакета: `tcp` или `tcpfwd`.

## 4. Корневая причина

Проект рос **по фичам, не по доменам**. Каждая новая способность (wrap, bond, vless, scan) приклеивалась соседней сущностью вместо встраивания в иерархию. Итог:
- термины перегружены (client/bond/mode/stream);
- CLI ≠ Go-имена;
- DTLS-as-obfuscation нигде не объяснён;
- captcha и proxy раздроблены без причины;
- два режима прокси не имеют общего родителя.

Первый рефакторинг (1–6) **вытаскивал** код из main, но не **перестраивал** домен. Сейчас main маленький, но домен такой же запутанный.

## 5. Целевая архитектура

```
cmd/
  vk-turn-client/main.go        # wiring only, ~150 строк
  vk-turn-server/main.go        # wiring only, ~100 строк

internal/
  logx/                         # slog-обёртка, заменяет isDebug+debugf
  config/                       # Parse*; группированные опции
  ctxutil/                      # closeWrite, pipe (data-copy)

  wire/
    srtpmimicry/                # бывш. internal/wrap
    bondframe/                  # бывш. internal/bond/frame.go

  transport/
    turndial/                   # бывш. internal/turnpipe
    dtlsdial/                   # есть
    kcptun/                     # бывш. tcputil; sync.Once для профиля

  proxy/
    engine/                     # общая часть (TURN×N + DTLS + creds + counters)
    udprelay/                   # бывш. proxy/udp; Handler
    tcpfwd/                     # бывш. proxy/vless; Handler
    bondclient/                 # бывш. internal/bond/client
    bondserver/                 # бывш. internal/bond/server

  netconn/                      # бывш. netadapt

  client/                       # КЛИЕНТ-ONLY (схлопывает client/internal)
    vkauth/                     # бывш. client/internal/vkauth
    captcha/
      auto/                     # бывш. client/internal/captcha
      manual/                   # бывш. client/manual_captcha.go (НОВЫЙ ПАКЕТ)
    browserprofile/             # бывш. client/internal/browserprofile
    dnsdial/                    # бывш. client/internal/dnsdial
    namegen/                    # бывш. client/internal/namegen
    ish/                        # бывш. internal/ish (явно клиент-only)

  server/
    bondregistry/               # бывш. internal/bond/server, если выносить globalBondRegistry чисто

  stats/                        # есть
```

### Принципы

1. **`cmd/<bin>/main.go` = wiring only.** Любая логика — в пакет.
2. **Одна `internal/` иерархия.** `internal/client/*` и `internal/server/*` — бинарь-специфика; `internal/{wire,transport,proxy}` — общее.
3. **Группировка по слою:** wire (байты на проводе) → transport (как открыть сокет) → proxy (что копировать) → client/server (роль).
4. **`logx.Logger` interface** заменяет `Deps{Debug bool, Debugf func}`. Один раз конфигурим в main.
5. **Глобалы умирают:** `globalBondRegistry` → локальная переменная в `cmd/vk-turn-server/main.go`. `isDebug` → logger level.
6. **Имена честные:**
   - `tcputil` → `kcptun`
   - `wrap` → `srtpmimicry`
   - `netadapt` → `netconn`
   - `turnpipe` → `turndial`
   - `vless` → `tcpfwd` (внутреннее; CLI остаётся `-vless`)
   - `bond` (codec) → `bondframe`
7. **`manual_captcha.go` → `internal/client/captcha/manual/`** как полноценный пакет с экспортируемым `Server`/`Solver`.
8. **`proxy.Engine` + `Handler` interface:** убирает дублирование udp/vless.
9. **Side-quests закрываются в этом этапе или не закрываются никогда.**

### Что НЕ делать
- `src/` — не Go-way, не вводить.
- `pkg/` — не вводить, наружу ничего публичного не отдаём.
- Wire-форматы / CLI флаги / behavior — заморожены.

## 10. Решения заказчика (зафиксировано 2026-05-19)

1. **libvkturn.so** — мобильная сборка, собирается вручную, переносится в приложение руками. CI не строит. Внешних `//export` в коде нет. **Решение:** `pkg/` не вводим, всё `internal/`. Пользователь после V2-1 пересоберёт с новым путём `./cmd/vk-turn-client` (или с buildmode-target, который он использует).
2. **Server-флаг `-vless-bond`** — **удаляем**. Autodetect по magic покрывает. Это **breaking CLI change на сервере** — фиксируем в release notes.
3. **`-no-dtls` (client `Direct`)** — **удаляем**. Помечен DO NOT USE, в проде не работает. Это **breaking CLI change на клиенте**.
4. **Server multi-tenant** (`notes/AUTH_PLAN.md`) — **план отменён**, вычёркиваем из контекста.
5. **TURN-scanner** (`client/scan*.go`) — открытый вопрос; кандидат `internal/client/turnscan/`. Решить при V2-5.

### Влияние на non-breaking гарантии

Пункт 8 («CLI флаги bit-exact») **корректируется**:
- Удаляются: `-no-dtls` (client), `-vless-bond` (server). Остальные CLI флаги — без изменений.
- Дефолты и поведение оставшихся флагов — bit-exact.
- Wire-форматы — bit-exact (без изменений).
