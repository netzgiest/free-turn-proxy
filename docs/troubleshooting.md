# Решение Проблем

- Сначала запускайте client, потом включайте WireGuard.
- WireGuard забирает весь трафик → пайп с `-debug 2>&1` и скриптом маршрутов обязателен: `scripts/routes.sh` / `scripts/routes.ps1` / `scripts/routes-macos.sh`.
- Маршруты не добавляются → убедитесь, что в команде есть `-debug 2>&1 |` перед скриптом.
- TCP до TURN не работает → попробуйте `-udp`.
- Соединение нестабильное → уменьшите `-n` (например, `-n 1`).
- VK часто требует captcha → `-manual-captcha`, затем обычный запуск.
- Клиент висит на получении TURN-данных → ссылка VK Calls умерла / звонок завершён для всех.
- Docker bridge mode → `CONNECT_ADDR=127.0.0.1:...` указывает внутрь контейнера. Используйте `--network host` или IP хоста.
- `-wrap` включён → ключ должен совпадать на клиенте и сервере.
- `VLESS_BOND=true` в docker → не ставьте, серверный флаг удалён в V2-0.
